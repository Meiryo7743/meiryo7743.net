フォントのイロハ

フォントの話をするにあたって、その基本事項を最初に共有しなければなりません。和文フォント、欧文フォントのそれぞれについて、本記事を深く理解する上で抑えておくべき点を 5 つ説明します。

和文フォントと欧文フォント

様々なフォントがこの世には存在します。その中には日本語文字を表示できるもの，半角英数字・記号しか対応していないものがあります。ものすごくザックリとした捉え方で，日本語文字に対応しているものを和文フォント，半角英数字・記号しか対応していないものは欧文フォントです。例えば，『メイリオ』は前者であり，『Times New Roman』は後者に該当します。

とはいえ，このような大雑把な区分け方だと厳密には誤解を招くものではあります。あくまでも本記事の理解に限った話であることに留意してください。

等幅とプロポーショナル

下図のように同じフォントサイズで同じ文字列であっても，フォントによっては，文字列の占める幅が異なることがあります。どの文字列でも 1 文字の幅が等しいフォントは等幅フォントで，逆に文字毎に幅が異なる場合はプロポーショナル（可変幅フォント）と言われます。

前者は文字幅が一定で印刷レイアウトを計算しやすく，昔のタイプライターが使われていた頃から利用されているようです。ただ，i と W が同じ幅であるように，文字によっては読みづらさを感じることもあります。また，括弧のような記号（約物）の場合は半角 1 文字分の余白が生まれ，文字同士の組合せによって，カーニングという文字間隔の調節を行わないと見栄えが悪くなることもあります。

一方で，後者は文字ごとに見やすい幅を採用するようにしており，普段手書きするような自然な感覚で文字を表示することができます。そのかたわら，文字のレイアウトを手動で調整することは難しくなるところもポイントとして挙げられます。

なお，等幅・可変幅のどちらのフォントも色々なところで実用されており，どちらが絶対に良いということではありません。用途に応じて，適切な使い分けを行うことが望ましいです。

書体

「書体」という言葉は，フォントの和訳語として一般に使用されることがあります。ただ，本記事では，明朝体，ゴシック体，丸ゴシック体，教科書体といったフォントの種類を総称する語として用いることにします。

ウェイト（太さの種類）

同じフォントでも，文字の太さに応じてバリエーションが存在することがあります。以下の図では，『メイリオ』を太字にした場合とそうでない場合を載せています。太さの点で見た目は異なりますが，どちらも同じフォントです。このような太さによるバリエーションの数のことをウェイトと呼称することにします。

互換性

今使っているデバイスで特定のフォントが表示できたり，反対にそのフォントが適用されなかったりといった現象が発生する要因は，フォントがインストールされているかどうかに依存します。Word がインストールされているデバイスでは docx ファイルを開けるけれども，未インストールの環境では開けずにエラーが出るなどするという場合と同じような感覚で捉えると良いでしょう。

もちろん，Windows でも macOS でも最初から使えるフォントは多数あります。そうしたフォントを互換性のあるフォントと定義しておくことにします。

すなわち，互換性のあるフォントを利用すれば，ある docx ファイルや pptx ファイルを他のデバイスで開いてもフォントが置き換わったり，レイアウトが崩壊したりせずに済むわけです。

おすすめの組み合わせ

前章が少し長くなりましたが、いよいよ個人的に推したい互換性重視で選ぶフォントの組み合わせを紹介いたしましょう。

『Segoe UI』と『BIZ UDゴシック』

欧文の『Arial』で和文は『ＭＳ ゴシック』，あるいは欧文も和文も『ＭＳ ゴシック』という昔から採用されがちなフォント設定の代わりとして用いてみてはどうでしょうか。互換性のあるフォントが乏しかった Windows 7 以前の時代において，『Arial』と『ＭＳ ゴシック』のコンビは最適解でした。しかし，和文フォントも充実している 2024 年現在，より読みやすくデザインされた互換性のあるゴシック体のフォントがあるのですから，採用しない理由がありません。

『BIZ UDゴシック』は，ゴシック体の UD フォント（ユニバーサルデザインフォント）の一種です。日本語文字のデザインは良いのですが，半角英数字の見た目はイマイチに映ります。ここに Windows の UI フォントとして採用されている『Segoe UI』と組み合わせることで，調和のある見た目が実現できます。

『Segoe UI』と『メイリオ』

先の『Segoe UI』と『BIZ UDゴシック』と同じ用途に適した組合せで，特に太字と細字による強弱を付けたい場合にピッタリです。Windows に標準搭載されている『BIZ UDゴシック』には太字のウェイトがありません。そこで，代わりに『メイリオ』を採用すると良い感じになります。

『Times New Roman』と『BIZ UD明朝 Medium』

これまで提示したものはゴシック体でしたが，それの明朝体版がこれです。Word でデフォルトのフォントとなっている『Century』と『游明朝』のコンビの代わりとして使ってみると良いかもしれません。

特に，『游明朝』だと細く，かすれた字のようで読みづらいと感じられる人にとっては，少し太めの『BIZ UD明朝 Medium』が最適です。これに合わせて，日本語文書でも採用ケースが多く，やはり太めの『Times New Roman』が良い感じに活きて来ることでしょう。

『Cambria』と『BIZ UD明朝 Medium』

欧文フォントに『Cambria』を使うのは，少し大胆な選択ではあります。ただ，Word や PowerPoint で数式を含む文書を作らなければならないときには，これを選んでも悪くないようにも見えます。

もちろん，TeX でおなじみの『Computer Modern』系統のフォントで組まれた数式の方が見栄えが良いのは間違いありません。それでも，『Times New Roman』と別の数式フォント（あるいは数式の画像）が混在するくらいなら，もういっそのこと『Cambria』と『Cambria Math』による統一美を選ぶと自然な仕上がりになるはずです。

おわりに

『ＭＳ 明朝』や『ＭＳ ゴシック』以外にも，デフォルトのフォントだけでより読みやすい良い組合せがあることをいくつか紹介しました。皆さんが文書作成をする際の参考になれば幸いです。
